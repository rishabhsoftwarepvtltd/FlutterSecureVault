## Responsible Disclosure
- Do not publicly disclose before a patch is released.
- Do not access or modify user data.
- Avoid exploitation beyond testing.

---

## Contributor Security Expectations
- Never commit secrets, API keys, tokens, or passwords.
- Validate and maintain dependencies.
- Check vulnerabilities (`flutter pub outdated`).
- Avoid unsafe practices such as insecure storage or ignoring SSL errors.
- Write tests for security-sensitive areas.
